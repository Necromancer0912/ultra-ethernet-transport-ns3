UET Transport Profiles

Three profiles define minimum required capabilities. Each maps to distinct AI/HPC use cases.

0
🤖

AI Base

Priority Level 0

Lowest cost, fundamental AI workloads. Supports CCL collectives (AllReduce, AllGather) and basic RDMA operations.

  • RDMA WRITE / READ
  • RUD + ROD + RUDI + UUD
  • CCL (AllReduce, AllGather)
  • PDC SYN + SACK bitmap
  • FI_HMEM (GPU memory)
  • FI_TAGGED send/recv
  • FI_ATOMIC operations
  • Deferrable send
  • SHMEM / MPI

AI Full

Priority Level 1

Complete AI feature set. Adds tagged-send rendezvous, deferrable sends for pipeline bubbles, and fetching atomics.

  • All AI Base features
  • FI_TAGGED (exact + wildcard)
  • FI_ATOMIC (fetch + non-fetch)
  • Deferrable Send
  • SHMEM put/get
  • FI_DIRECTED_RECV
  • Rendezvous protocol
  • MPI collectives

HPC

Priority Level 2

High Performance Computing. All AI Full features except deferrable send, plus MPI collectives and SHMEM symmetric heaps.

  • All AI Full features
  • MPI_Allreduce / Scatter
  • SHMEM symmetric heaps
  • Non-fetching atomics
  • ROD ordered delivery
  • Halo exchange
  • Deferrable send

Profile Interoperability (§2.2.2.1): AI Base is a subset of both AI Full and HPC.
